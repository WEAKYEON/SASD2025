# HW3: C# Class Analysis from .NET Based APIs

## a) Utility Class
**Class Name:** `System.Math`

**การวิเคราะห์:**
`System.Math` จัดเป็น **Utility Class** เนื่องจากเหตุผลดังนี้:
1.  **Static Keyword:** คลาสนี้ถูกประกาศเป็น `public static class` ซึ่งหมายความว่าเราไม่สามารถสร้าง Instance (Object) ของคลาสนี้ได้ (ไม่สามารถใช้คำสั่ง `new Math()` ได้)
2.  **Stateless:** ภายในคลาสไม่มีการเก็บข้อมูลสถานะ (State) ของวัตถุ มีเพียงค่าคงที่ (Constants) เช่น `PI`, `E` และ Method ที่ใช้คำนวณทางคณิตศาสตร์
3.  **Pure Functions:** Method ต่างๆ เช่น `Math.Abs()`, `Math.Sin()`, `Math.Max()` ทำงานในลักษณะรับค่าเข้าไปประมวลผลและส่งผลลัพธ์ออกมา โดยไม่ขึ้นกับสถานะภายในใดๆ

## b) Mutable Class
**Class Name:** `System.Text.StringBuilder`

**การวิเคราะห์:**
`System.Text.StringBuilder` จัดเป็น **Mutable Class** เนื่องจากเหตุผลดังนี้:
1.  **Internal State Modification:** เมื่อเราเรียกใช้ Method เช่น `.Append()`, `.Insert()`, หรือ `.Replace()` คลาสนี้จะทำการแก้ไขข้อมูลภายใน Object ตัวเดิมโดยตรง ไม่ได้สร้าง Object ใหม่
2.  **Writable Properties:** มี Property ที่สามารถแก้ไขค่าได้หลังจากสร้าง Object แล้ว เช่น `Capacity` หรือ `Length` (ในบางเงื่อนไข) ซึ่งทำให้สถานะของ Object เปลี่ยนแปลงไปได้ตลอดเวลาที่มีการใช้งาน

## c) Immutable Class
**Class Name:** `System.String`

**การวิเคราะห์:**
`System.String` จัดเป็น **Immutable Class** เนื่องจากเหตุผลดังนี้:
1.  **Read-Only State:** เมื่อสร้าง string ขึ้นมาแล้ว ข้อมูลตัวอักษรภายในจะไม่สามารถถูกแก้ไขได้เลย
2.  **Return New Instance:** การเรียกใช้ Method ที่ดูเหมือนจะแก้ไขข้อความ เช่น `.ToUpper()`, `.Substring()`, หรือ `.Replace()` ความจริงแล้ว Method เหล่านี้ไม่ได้แก้ที่ตัวแปรเดิม แต่จะทำการสร้าง string ก้อนใหม่ (New Instance) ในหน่วยความจำและส่งค่ากลับมา
3.  **Thread Safety:** เนื่องจากค่าภายในไม่สามารถเปลี่ยนได้ จึงทำให้ปลอดภัยเมื่อถูกเรียกใช้จากหลาย Thread พร้อมกัน (โดยไม่ต้อง Lock)
